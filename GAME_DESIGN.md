# 좀비 헌팅 RPG — 게임 디자인 스펙 (GDD v1)

> 이 문서는 **모든 숫자·공식의 단일 원본(Single Source of Truth)** 이다.
> 현재 운영 중인 `game.html`(game.js)에서 실제 값을 추출해 작성했다. 재구성 시 클라이언트·서버가 **이 문서/`balance.json` 하나만** 참조한다.
> 표기: `lv`=레벨, `region`=현재 지역, `stage`=현재 스테이지(1~10), `rebirth`=환생 횟수.

---

## 1. 개요

- **장르**: 방치형(Idle/Incremental) 수집 RPG + 경쟁 랭킹
- **플랫폼**: 모바일 웹(세로 1열), 강남·논현 카카오톡 커뮤니티 공유용
- **핵심 판타지**: 좀비가 점령한 강남 밤거리를 생존자가 자동으로 사냥하며 성장·수집·경쟁
- **디자인 기둥**
  1. **방치 기본 + 능동 가속**: 접속만 해도 자동 사냥으로 성장, 뽑기·강화·아레나로 가속
  2. **눈에 보이는 성장**: 장비가 캐릭터 외형에 반영(페이퍼돌), 전투력/랭킹으로 즉시 확인
  3. **커뮤니티 경쟁**: 단톡 멤버끼리 전투력·환생·도감·아레나 6개 보드로 경쟁
  4. **가벼운 세션**: 한 번에 몇 분, 자주 들어오게 만드는 접속 보상 구조

## 2. 핵심 루프

```
자동 사냥(전투) → 골드·경험치·강화석·전리품 획득
   → 레벨업(전투력↑, 스킬포인트) · 전리품 뽑기(장비)
   → 장비 장착·강화(전투력↑) · 스킬 학습 · 도감 수집
   → 더 강한 지역/보스 돌파(스테이지 진행) → 랭킹 상승
   → (레벨 30+) 환생: 초기화하되 영구 보너스 + 도감/최고기록 유지
```

부가 루프: 출석·보급상자·일일미션(접속 동기) / 아레나(PvP 점수).

## 3. 진행 시스템

### 3.1 지역 & 스테이지
- 한 지역 = **스테이지 1~10**. 각 스테이지에서 `STAGE_KILLS = 8`마리 처치 시 다음 스테이지.
- 스테이지 10 = **보스**. 보스 처치 시 `region++`, 스테이지 1로. (지역은 **무한 진행**)
- **최고기록**: 현재 진행도 `(region-1)*10 + stage` 가 최고치를 갱신하면 `bestRegion/bestStage` 저장. 환생해도 **보존**.
- 지역 이름/테마/적은 7종이 순환(아래 9장). region이 7을 넘으면 인덱스 순환.

### 3.2 레벨 & 경험치
- 필요 경험치: `expNeed = round(80 * lv^1.35)`
- 레벨업 시: **스킬포인트 +1**, **젬 +2**

### 3.3 환생 (Rebirth, 프레스티지)
- 조건: `lv >= 30 + rebirth*10`
- 처리: 상태를 초기화하되 **도감·환생횟수·최고기록(region/stage) 유지**, 젬 +30, 시작 보너스(특성P=rebirth, 골드=800×rebirth, 강화석=50×rebirth)
- **영구 보너스**
  | 항목 | 공식 | 회당 |
  |---|---|---|
  | 전투력 배수 | `1 + rebirth*0.25` | +25% |
  | 골드 획득 | `1 + rebirth*0.18` | +18% |
  | 방치 사냥 시간 한도 | `8 + rebirth` 시간 | +1h |
  | 보너스 특성P | 환생 시 `rebirth` 만큼 즉시 지급 | — |

## 4. 전투 (자동)

- **자동 진행**: 사냥 탭에서 적이 스폰되고 캐릭터가 자동 공격.
- **내 피해**: `dmg = max(3, round(power * 0.8))`, 크리티컬 시 `×1.8`
- **크리티컬 확률**: `critChance = 0.05`  ⚠️ (현재 코드: `0.05 + luck*0.0` → **행운 스킬이 0으로 곱해져 크리에 영향 없음**. 12장 이슈 참고)
- **적 체력**: `enemyHP = round((18 + region*22 + stage*6) * (보스?6:1) * (1 + bestRegion*0.05))`
- **콤보**: 연속 처치 시 `combo+1`, 유지창 `2800ms`. 콤보 보너스 = 골드 `×(1 + min(combo,25)*0.04)` (최대 +100%)

### 전투력 공식 (★ 가장 중요 — 클라/서버 동일해야 함)
```
power = floor(
  (12 + lv*6 + bestStageEff*2 + gearPow)
  * (1 + atk_lv*0.06)
  * (1 + dexBonusPct/100)
  * (1 + rebirth*0.25)
)
```
- `bestStageEff = min((bestRegion-1)*10 + bestStage, 100 + lv*14 + rebirth*100)` (진행도 반영 + 어뷰징 상한)
- `gearPow = Σ itemPow(장착장비)`,  `itemPow = round(item.power * (1 + plus*0.13))`
- `dexBonusPct = dexCount * 1.2 * (1 + collect_lv*0.25)`

## 5. 전리품 / 가챠

### 5.1 등급 (RAR)
| idx | 등급 | 가중치(w) | 확률(근사) | 파워배수(mul) | 색 |
|---|---|---|---|---|---|
| 0 | 일반 | 62 | 62.11% | ×1 | #9aa0aa |
| 1 | 고급 | 27 | 27.05% | ×1.7 | #46c46e |
| 2 | 희귀 | 9 | 9.02% | ×3.0 | #3d8bf0 |
| 3 | 영웅 | 1.7 | 1.703% | ×5.5 | #b06ff0 |
| 4 | 전설 | 0.12 | 0.1202% (~1/833) | ×13 | #e6c964 |
| 5 | 신화 | 0.00001 | ~0.00001% (~1/1000만) | ×25 | #ff7a59 |

*총 가중치 99.82001. 확률 = w / 99.82001.*

- **천장(Pity)**: `PITY = 70`. 영웅(이상) 없이 70회 누적 시 다음은 **영웅 확정**. (전설·신화는 천장으로 안 나옴 — 극악 유지)
  - pity는 영웅(idx≥3) 획득 시 0으로 리셋, 그 외엔 +1.
- **행운(luck) 영향**: 희귀 가중치 `×(1+luck*0.04)`, 영웅 `×(1+luck*0.03)` 만. (다른 등급엔 영향 없음)

### 5.2 슬롯 추첨 (SLOTW) & 아이템 파워
- 드랍 슬롯 가중치: `weapon 40 / top 20 / head 16 / legs 9 / cape 15` (합 100)
- 아이템 파워: `power = max(1, round(PBASE[slot] * RAR[rarity].mul * rand(0.9~1.2) * (1 + lv*0.07 + region*0.25)))`
- 슬롯별 기본치 `PBASE`: `weapon 12 / top 7 / head 6 / legs 7 / cape 5`

### 5.3 뽑기 비용
| 동작 | 비용 | 비고 |
|---|---|---|
| 1회 뽑기 | 행동력 `3` (PULL1) | |
| 10연 뽑기 | 행동력 `27` (PULL10), 부족 시 **젬 20**으로 대체 | |

### 5.4 도감 (Codex)
- 아이템을 **처음 획득**하면 도감에 등록(`slot:type` 키, 최고 등급 기록).
- 총 **100종**: weapon 16 / top 32 / head 36 / legs 13 / cape 3.
- 도감 수집량 `dexCount`이 전투력에 직접 기여(4장 `dexBonusPct`).

## 6. 장비

- **슬롯 5종**: weapon / top / head / legs / cape (각 1개 장착, 외형 반영)
- **강화(Enhance)**: 최대 `ENH_MAX = 15`
  - 비용: `gold = round(40 * 1.5^plus * (rarity+1))`, `stone = round(2 + plus*1.5 + rarity)`
  - 효과: `itemPow = round(power * (1 + plus*0.13))` (강화당 +13%p 기준)
- **분해(Salvage)**: `강화석 = round((rarity+1) * (2+plus) * 1.5 * (1 + wealth_lv*0.08))`
- 자동 분해(`autoSalvage`) 등급 임계값 설정 가능(기본 0=사용 안 함).

## 7. 스킬 (특성) — 5종

| id | 이름 | 효과 | 적용 공식 |
|---|---|---|---|
| atk | 전투 숙련 | 전투력 +6%/Lv | `×(1+atk*0.06)` |
| luck | 행운 | 전리품 등급↑/Lv | 희귀×(1+0.04/Lv), 영웅×(1+0.03/Lv) ⚠️크리 미반영(이슈) |
| wealth | 수색 | 골드·강화석 +8%/Lv | `wealthMul=(1+wealth*0.08)*(1+rebirth*0.18)` |
| haste | 신속 | 행동력 회복·공속↑/Lv | 행동력 회복 간격 `×(1-haste*0.05)` |
| collect | 수집가 | 도감 보너스 +25%/Lv | `dexBonus mul ×(1+collect*0.25)` |

- 학습 비용: `pt = 1`, `gold = round(120 * 1.4^현재Lv)`
- 스킬포인트 출처: 레벨업(+1), 환생(rebirth 만큼).

## 8. 재화 & 경제

### 8.1 화폐
| 화폐 | 용도 | 시작값 |
|---|---|---|
| 골드 🪙 | 강화·스킬 학습 | 60 |
| 강화석 💠 | 강화 | 0 |
| 젬 💎 | 10연 대체·각종 보상 | 30 |
| 행동력(에너지) ⚡ | 뽑기 | MAX=30(시작 풀) |

### 8.2 행동력(에너지)
- 최대 `MAXE = 30`. 회복: `REGEN_MS = 80000ms`마다 +1 (신속 스킬로 단축: `×(1-haste*0.05)`)
- **전투 보너스**: 5마리 처치마다(`keCtr`) +1 행동력(최대까지)

### 8.3 소스/싱크 요약
| 구분 | 골드 | 강화석 | 젬 | 행동력 |
|---|---|---|---|---|
| **소스** | 처치, 보급상자, 출석, 일일미션 | 처치(보스/확률), 분해, 보급상자, 출석, 일일미션 | 레벨업(+2), 아레나 승(+4), 출석, 일일미션, 보급상자 | 시간회복, 처치(5킬당+1) |
| **싱크** | 강화, 스킬 학습 | 강화 | 10연 대체(20) | 뽑기(1회3/10연27) |

### 8.4 처치 보상 (m = 보스8 / 일반1)
- 골드: `round((5 + region*4 + stage) * rand(0.85~1.15) * m * wealthMul * comboMul)`
- 경험치: `round((4 + region*2.2 + stage*0.6) * rand(0.85~1.15) * m)`
- 강화석: 보스 `round((6+region*2)*wealthMul)` / 일반 25%확률 `round(1+region*0.5)`
- 전리품 드랍: 보스 `0.9`, 일반 `0.13 + luck*0.01`

## 12. 알려진 이슈 / 리밸런스 후보 (재구성 시 결정)

1. **크리티컬-행운 버그**: `critChance = 0.05 + luck*0.0` → 행운이 크리에 전혀 반영 안 됨. → 행운에 크리 확률을 묶거나, 행운 설명을 정정.
2. **가챠/재화 서버 미검증**: 뽑기·강화·구매 차감이 클라에만 있어 연타/조작 여지. → **서버 RPC로 원자적 처리**(무한 뽑기류 차단).
3. **신화 ~1/1000만**: 사실상 획득 불가(연출용). 의도면 유지, 아니면 상향.
4. **진행도→전투력 비중**: 최근 `bestStage*2` 추가. 방치형 밸런스(장비 vs 진행) 재튜닝 필요.
5. **봇 랭킹 ↔ 실서버 랭킹 혼재**: 재구성 시 **실데이터 단일 경로**로 단순화(봇은 초기 부트스트랩용으로만, 혹은 제거).
6. **지역 무한 진행**: REGION_NAME/THEME 순환 인덱싱 규칙 명문화(`index % 7`).
7. **신원/저장 구조**: 익명+localStorage → **카카오 로그인 + 서버 권위 클라우드 세이브**로 전환(별도 결정 완료).

## 13. 데이터 주도 설계 (재구성 핵심 원칙)

- 위 모든 숫자는 `balance.json` 한 파일에 모은다. **클라이언트와 서버(전투력 계산·보상 검증)가 같은 JSON을 import** 한다. → 공식이 두 곳에 갈라져 어긋나는 문제 원천 차단.
- 전투력·보상·가챠 결과처럼 **랭킹/경제에 영향 주는 계산은 서버가 권위**를 가진다(클라는 표시·연출).
- 게임 상태(S 전체)는 **서버에 JSONB로 저장**, 로그인 신원(카카오 uid) 기준. localStorage는 오프라인 캐시로만.
